# 产品需求文档 (PRD) - FlexQuant Strategies

## 1. 项目愿景
FlexQuant 旨在为个人投资者提供一个轻量级、自动化且易于扩展的 A 股量化交易分析框架。通过集成多源数据、自动化调度、可视化回测及实时通知推送，帮助用户高效地发现并验证基于"涨停模式"的交易机会。

## 2. 目标用户
- 对 A 股量化交易感兴趣的技术型投资者。
- 希望实现选股逻辑自动化、减少手动筛选时间的交易者。
- 需要简单回测引擎来验证策略有效性的开发者。

## 3. 核心功能模块

### 3.1 自动化数据中心
- **多源获取**: 集成 Akshare, Baostock, Tushare。
- **高效存储**: 全面采用 Parquet 格式，利用 Snappy 压缩提升性能。
- **生命周期管理**: 强制 180 交易日滑动窗口，确保存储空间可控。
- **并发下载**: 多线程加速全量数据更新。

### 3.2 策略分析引擎
- **选股逻辑**: 专注于"涨停模式"（如 N 日内涨停次数、回调幅度）。
- **动态评分**: 结合价格、市值、技术指标（MA, RSI）及盈利状况进行多维度打分。
- **Top 50 机制**: 自动筛选得分最高的股票进入观察池。

### 3.3 回测验证系统
- **可视化回测**: 在 Web 端直接运行策略，生成资金曲线和最大回撤。
- **交易明细**: 详细记录每一笔买入、卖出及盈亏情况。
- **自动预计算**: 回测时自动生成技术指标，确保与实盘筛选逻辑一致。

### 3.4 Web 交互终端 (UI)
- **图形化操作**: 无需编写代码即可调整超参、更新数据、查看 K 线。
- **中文支持**: 全界面汉化，符合国内用户习惯。

### 3.5 自动化通知
- **推送集成**: 支持飞书、钉钉机器人 Webhook。
- **每日任务**: 选股完成、回测结束及异常报错自动推送报告。

## 4. 核心业务流程
1. **数据抓取**: 每日收盘后（如 20:00）自动触发。
2. **特征计算**: 更新 MA、RSI、涨停计数等。
3. **选股排序**: 根据当日数据生成 Top 50 推荐列表。
4. **自动回测**: 对推荐列表执行历史验证。
5. **推送结果**: 将推荐清单及回测简报发送至社交软件。

## 5. 验收标准
- 每日任务能在预定时间稳定执行。
- Web 界面能正确展示至少过去 50 条回测交易记录。
- 数据存储目录大小不应随时间无限增长（受滑动窗口控制）。

## 6. 策略逻辑定义

### 6.1 核心策略理念
FlexQuant 策略基于"涨停模式"选股，通过捕捉具有强势特征的股票，结合技术面和基本面分析，寻找具有上涨潜力的投资机会。

### 6.2 买入条件
策略采用多因子筛选机制，只有满足以下所有条件的股票才会被考虑买入：

#### 6.2.1 基础条件
1. **价格区间**: 股价在 [min_price, max_price] 范围内
2. **市值限制**: 总市值不超过 max_market_cap 亿元
3. **盈利要求**: 如果启用 require_profit，则要求净利润为正
4. **涨停次数**: 在 limit_up_period 日内，涨停次数在 [min_limit_up_count, max_limit_up_count] 范围内
5. **涨幅控制**: N 日内涨幅不超过 max_price_increase

#### 6.2.2 技术面条件
1. **均线多头**: MA5 > MA20，表示短期趋势向上
2. **RSI指标**: RSI < 75，避免超买状态
3. **成交量**: 成交量不低于 min_volume 万股

### 6.3 卖出条件
采用多重止盈止损机制，确保风险控制：

#### 6.3.1 止盈条件
1. **目标收益率**: 达到 profit_target 时止盈（默认 20%）
2. **技术面恶化**: MA5 < MA20，趋势反转
3. **RSI超买**: RSI > 80，强烈超买信号

#### 6.3.2 止损条件
1. **固定止损**: 亏损达到 stop_loss_ratio 时止损（默认 10%）
2. **时间止损**: 持股时间超过 holding_period_limit 天（默认 30 天）

### 6.4 持仓管理
- **最大持仓数**: 同时持有的股票数量不超过 max_stocks
- **仓位管理**: 采用等权重分配，每只股票仓位相对均衡
- **资金管理**: 确保有足够的可用资金进行新的买入操作

## 7. 因子定义

### 7.1 价格因子
- **当前价格**: 股票最新收盘价
- **价格位置**: 相对历史价格区间的位置
- **价格波动**: 近期价格波动率

### 7.2 涨停因子
- **涨停次数**: N 日内涨停次数
- **涨停间隔**: 连续涨停或间隔涨停模式
- **涨停强度**: 涨停时的成交量、换手率等

### 7.3 技术因子
- **移动平均线**: MA5, MA20, MA60
- **均线排列**: 多头排列（MA5 > MA20 > MA60）得分更高
- **相对强弱指标**: RSI，避免超买超卖
- **趋势强度**: 价格趋势的持续性

### 7.4 基本面因子
- **市值**: 总市值，偏好中小盘股
- **盈利能力**: 净利润、毛利率等
- **成长性**: 营收增长率、利润增长率

### 7.5 成交量因子
- **成交量**: 绝对成交量水平
- **量价关系**: 价格上涨时的成交量放大
- **换手率**: 股票的活跃程度

## 8. 评分机制

### 8.1 评分权重
策略采用多因子加权评分机制，各因子权重如下：

| 因子类别 | 权重 | 说明 |
|---------|------|------|
| 价格因子 | 20% | 价格越低，评分越高 |
| 涨停因子 | 30% | 涨停次数适中，评分最高 |
| 技术因子 | 25% | 均线多头排列、RSI适中得分高 |
| 基本面因子 | 15% | 市值适中、盈利能力强得分高 |
| 成交量因子 | 10% | 成交量活跃得分高 |

### 8.2 评分计算
总评分 = Σ(因子得分 × 因子权重)

评分范围：0-100 分，分数越高表示投资价值越大。

### 8.3 排序规则
- 按总评分降序排列
- 评分相同时，优先选择涨停次数较多的股票
- 仍然相同时，优先选择市值较小的股票

## 9. 风控要求

### 9.1 单股票风险控制
- **最大单股仓位**: 不超过总资金的 20%
- **最大持仓时间**: 不超过 holding_period_limit 天
- **止损比例**: 单股最大亏损不超过 stop_loss_ratio

### 9.2 组合风险控制
- **最大持仓数量**: 不超过 max_stocks 只股票
- **行业集中度**: 避免单一行业持仓过于集中
- **相关性控制**: 避免持有高度相关的股票

### 9.3 市场风险控制
- **大盘风险**: 大盘下跌时减少仓位
- **波动率控制**: 市场波动率过高时降低仓位
- **流动性风险**: 避免持有流动性差的股票

### 9.4 数据风险控制
- **数据质量**: 确保数据来源可靠、数据完整
- **数据时效**: 使用最新的市场数据
- **异常检测**: 检测并处理异常数据

### 9.5 系统风险控制
- **异常监控**: 监控系统运行状态，及时发现异常
- **容错机制**: 单个股票出错不影响整体运行
- **日志记录**: 详细记录系统运行日志，便于问题排查

## 10. 策略参数配置

### 10.1 核心参数
```yaml
strategy:
  stock_selection:
    # 价格区间
    min_price: 10.0          # 最低价格（元）
    max_price: 50.0          # 最高价格（元）
    
    # 市值限制
    max_market_cap: 100.0    # 最大市值（亿元）
    
    # 涨停参数
    min_limit_up_count: 1    # 最小涨停次数
    max_limit_up_count: 5    # 最大涨停次数
    limit_up_period: 20      # 涨停统计周期（天）
    limit_up_threshold: 9.8  # 涨停阈值（%）
    
    # 涨幅控制
    max_price_increase: 30.0 # 最大价格涨幅（%）
    
    # 持仓管理
    holding_period_limit: 30 # 最大持股天数
    profit_target: 20.0      # 目标收益率（%）
    stop_loss_ratio: 10.0   # 止损比例（%）
    
    # 其他条件
    min_volume: 100.0        # 最低成交量（万股）
    require_profit: true    # 是否要求正盈利
    concept_weight: 0.3      # 概念叠加权重
```

### 10.2 参数调优建议
- **保守型**: 提高价格下限、降低市值上限、减少涨停次数要求
- **激进型**: 降低价格下限、提高市值上限、增加涨停次数要求
- **平衡型**: 使用默认参数配置

## 11. 策略评估指标

### 11.1 收益指标
- **总收益率**: 策略总收益 / 初始资金
- **年化收益率**: 年化后的收益率
- **超额收益**: 相对基准指数的超额收益

### 11.2 风险指标
- **最大回撤**: 策略最大亏损幅度
- **夏普比率**: 风险调整后收益
- **波动率**: 收益率的标准差

### 11.3 交易指标
- **胜率**: 盈利交易占比
- **盈亏比**: 平均盈利 / 平均亏损
- **交易频率**: 单位时间内的交易次数

### 11.4 持仓指标
- **平均持仓时间**: 股票平均持有天数
- **持仓周转率**: 持仓更换频率
- **集中度**: 前十大持仓占比

### 12.1 短期优化
- **因子优化**: 调整因子权重，提高选股准确性

### 12.2 中期优化
- **多策略组合**: 结合多种策略，降低单一策略风险
- **机器学习**: 引入机器学习方法，提高预测准确性
- **动态调整**: 根据市场状态动态调整策略参数

### 12.3 长期优化
- **深度学习**: 探索深度学习在策略中的应用
- **另类数据**: 引入另类数据源，丰富策略维度
- **实时交易**: 从选股策略升级为实时交易策略
